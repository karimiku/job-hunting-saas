# UX 再設計（β → リリース準備）

2026-07 実施。「UIは綺麗だが、就活生が何をすればいいか分からない」という課題に対する、ペルソナ・情報設計・画面の見直し。

## 1. ペルソナの再定義

### プライマリ: ミナミ（大学3年・6月、就活立ち上げ期）
- マイナビ / ONE CAREER / OfferBox / 企業マイページを併用。サマーインターン応募 8〜10社から始まり、本選考期には 25社前後を並行。
- Notion・スプレッドシートでの管理を試みて 2週間で破綻した経験がある。
- 移動中はスマホ、ES を書くときだけ PC。
- **困りごと（優先順）**:
  1. 「今日・今週、何をしないといけないか」が分からない（締切漏れが怖い）
  2. 「どの会社がどこまで進んでいるか」を一覧できない
  3. 管理ツールへの入力が面倒で続かない

### セカンダリ: 選考中盤のユーザー
- 10社以上が同時進行。面接日程・ES 締切・Webテスト期限が混在。
- ホームを開いた瞬間に「期限が過ぎているもの・今日のもの」だけ知りたい。

### 設計原則
1. **迷ったらホームに戻れば次の一手が分かる**（時間軸を最優先: 期限切れ → 今日 → 今週）
2. **専門用語を使わない**。「Entry」→「応募先」。データモデル名・ルート名は変えず、表示ラベルのみ日本語化。
3. **入力は 30秒以内**。会社名だけで登録できる。タスクは定型チップから 1タップ。
4. **初回ログインから 3分で「使い方が分かって、1社登録できている」**。

## 2. 情報設計（IA）の変更

| 旧ナビ | 新ナビ | 変更点 |
| --- | --- | --- |
| ホーム | ホーム | 時間軸ファースト（期限切れ/今日/今週）+ はじめかたガイド |
| Entry | 応募先 | ラベル変更。一覧⇄ボードのビュー切替タブを追加 |
| カンバン | ボード | 応募先のビューの一つとして位置づけ、切替タブで往来 |
| タスク | タスク | 定型テンプレチップ追加 |
| 保存箱 | 保存箱 | 説明文の平易化 |

- ルート（/entry, /kanban 等）と API・データモデルは変更しない。表示層のみ。
- /entry と /kanban のページ上部に共通の「一覧 | ボード」切替を置き、同じデータの別ビューであることを明示する。

## 3. 画面ごとの改善

### オンボーディング（全面刷新）
- 旧: 雰囲気コピー 3枚（機能説明なし）→ ダッシュボードに放り出す。
- 新: 「このサービスで何ができるか」を 3 ステップの実画面ベースで説明:
  1. 応募先を登録する（会社名だけで OK）
  2. 選考が進んだらカードを動かす（ボード）
  3. 締切・面接をタスクにして、ホームで今日やることを見る
- 最終 CTA は「最初の応募先を登録する」→ /entry/new。スキップ導線（ホームへ）も残す。

### ホーム
- **はじめかたガイド**（応募先 0 件のとき）: 3 ステップのチェックリストカード。完了状態はデータから判定（応募先≥1 / タスク≥1 / 完了タスク≥1）。
- **今日のタスク → 直近のタスク**: 期限切れを赤系で強調し「〜日超過」を表示。
- 次にやること（NextAction）は維持しつつ、文言を平易化。

### 応募先（Entry）
- 一覧/ボードのビュー切替タブ。
- 新規フォーム: ラベルを平易化（「応募の種類」「どこで見つけた？」）、選考フローの説明追加。作成後は **詳細ページへ遷移**し、「次は締切をタスクに追加」を促す。

### タスク
- 追加フォームに就活定型テンプレチップ（ES提出 / Webテスト / 適性検査 / 一次面接 / 二次面接 / 最終面接 / 説明会 / お礼メール）。タップでタイトルと種類が入る。
- Entry 詳細内のタスクフォームにも同チップを設置。

### 全体の可読性
- 8〜12px に密集していた文字サイズを一段引き上げ（最小 10px、本文 12〜13px）。タップ領域は既存を維持。
